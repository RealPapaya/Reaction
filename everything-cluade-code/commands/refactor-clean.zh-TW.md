# Refactor Clean

透過測試驗證安全地識別並移除死碼：

1. 執行死碼分析工具：
   - knip: 找出未使用的匯出與檔案
   - depcheck: 找出未使用的依賴
   - ts-prune: 找出未使用的 TypeScript 匯出

2. 在 .reports/dead-code-analysis.md 產生全面報告

3. 依嚴重性分類發現：
   - SAFE (安全): 測試檔案、未使用的工具
   - CAUTION (謹慎): API routes、組件
   - DANGER (危險): 設定檔、主要進入點

4. 僅提出安全的刪除建議

5. 在每次刪除之前：
   - 執行完整測試套件
   - 驗證測試通過
   - 套用變更
   - 重新執行測試
   - 如果測試失敗則回滾

6. 顯示已清理項目的摘要

絕不在未先執行測試的情況下刪除程式碼！
